# 中文指代消解与句子改写系统实现计划

## 1. 项目简介

本项目拟实现一个面向中文文本的指代消解与句子改写系统。系统输入一段中文文本，自动识别其中的代词或指代表达，判断其对应的先行实体，并生成更明确、可读性更强的改写文本。

示例：

```text
输入：小明把书给了小红，因为她需要它。
输出：小明把书给了小红，因为小红需要书。
```

项目重点不在于训练一个大型模型，而在于完整展示 NLP 项目的流程：文本预处理、候选实体抽取、指代关系判断、文本改写、可视化展示和实验评估。

## 2. 项目目标

本项目计划实现以下功能：

1. 支持用户输入中文文本。
2. 自动识别文本中的实体、名词短语和代词。
3. 为每个代词生成候选先行词。
4. 使用规则方法或规则结合语义模型的方法判断指代关系。
5. 输出指代消解结果，例如“她 -> 小红”。
6. 根据指代结果生成改写文本。
7. 使用 Streamlit 构建可视化 Demo 页面。
8. 构建小规模测试集，并进行实验评估。

## 3. 技术栈

### 3.1 编程语言

- Python

### 3.2 中文 NLP 工具

优先选择：

- HanLP

备选方案：

- LTP
- jieba + 自定义规则

HanLP 主要用于完成中文分词、词性标注、命名实体识别和依存句法分析，为后续指代消解提供基础语言信息。

### 3.3 语义匹配模型

可选使用：

- transformers
- Chinese-BERT
- Chinese-MacBERT
- sentence-transformers

如果时间有限，可以先实现纯规则版本；如果时间充足，再加入 BERT 类模型计算候选实体与代词所在上下文的语义匹配分数。

### 3.4 可视化框架

- Streamlit

Streamlit 用于构建网页 Demo，包括文本输入、原文高亮、指代结果展示、候选实体分数展示和改写文本展示。

### 3.5 实验评估

- scikit-learn
- pandas
- matplotlib 或 Streamlit 内置图表

用于计算 Accuracy、Precision、Recall、F1 等指标，并展示实验结果。

## 5. 核心模块设计

### 5.1 文本预处理模块

文件建议：`src/preprocess.py`

主要功能：

- 对输入文本进行分句。
- 调用 HanLP 进行分词。
- 获取词性标注结果。
- 获取命名实体识别结果。
- 获取依存句法分析结果。

输出示例：

```json
{
  "tokens": ["小明", "把", "书", "给", "了", "小红", "，", "因为", "她", "需要", "它", "。"],
  "pos": ["nr", "p", "n", "v", "u", "nr", "w", "c", "r", "v", "r", "w"],
  "entities": [
    {"text": "小明", "type": "PERSON", "start": 0, "end": 2},
    {"text": "小红", "type": "PERSON", "start": 7, "end": 9}
  ]
}
```

### 5.2 Mention 抽取模块

文件建议：`src/mention_extractor.py`

主要功能：

- 抽取人物、组织、地点等命名实体。
- 抽取普通名词作为候选实体，例如“书”“公司”“产品”。
- 抽取代词，例如“他”“她”“它”“他们”“该公司”“这个产品”。

候选 mention 类型：

- 人名
- 机构名
- 地名
- 普通名词
- 人称代词
- 指示代词

### 5.3 指代关系判断模块

文件建议：`src/resolver.py`

第一阶段建议实现规则方法：

1. 距离规则：优先选择离代词更近的候选实体。
2. 类型规则：“他 / 她”优先匹配人物实体。
3. 语义规则：“它”优先匹配物体、产品、事件等非人物实体。
4. 位置规则：候选先行词通常出现在代词之前。
5. 句法规则：优先考虑主语、宾语位置上的实体。

第二阶段可加入语义匹配模型：

1. 构造代词上下文。
2. 构造候选实体上下文。
3. 使用 BERT 或 sentence-transformers 计算相似度。
4. 将规则分数与语义分数加权融合。

最终打分公式可设计为：

```text
final_score = 0.5 * rule_score + 0.5 * semantic_score
```

### 5.4 文本改写模块

文件建议：`src/rewriter.py`

主要功能：

- 根据指代结果替换代词。
- 保持原句基本结构不变。
- 处理简单重复和标点问题。

示例：

```text
原文：小明把书给了小红，因为她需要它。
关系：她 -> 小红，它 -> 书
改写：小明把书给了小红，因为小红需要书。
```

### 5.5 可视化模块

文件建议：`app.py`

页面功能：

- 文本输入框。
- “开始分析”按钮。
- 原文高亮展示。
- 指代关系表格。
- 候选实体得分表格。
- 改写文本展示。
- 示例文本快速选择。

页面展示内容建议：

```text
输入文本：
小明把书给了小红，因为她需要它。

识别结果：
实体：小明、小红、书
代词：她、它

指代关系：
她 -> 小红
它 -> 书

改写结果：
小明把书给了小红，因为小红需要书。
```

## 6. 数据集设计

由于期末作业规模有限，可以自建小型数据集。

建议规模：

- 训练或开发样本：50 条
- 测试样本：30 条
- 展示样本：10 条

样本类型：

1. 人物指代：

```text
小明见到了小红，她正在图书馆看书。
```

2. 物体指代：

```text
小红买了一本书，她很喜欢它。
```

3. 组织指代：

```text
华为发布了新手机，该公司表示销量很好。
```

4. 多候选实体：

```text
小明把礼物送给小红，因为她今天过生日。
```

5. 歧义样本：

```text
小明告诉小刚，他已经完成了作业。
```

数据标注格式建议：

```json
{
  "text": "小明把书给了小红，因为她需要它。",
  "coreference": [
    {"pronoun": "她", "antecedent": "小红"},
    {"pronoun": "它", "antecedent": "书"}
  ],
  "rewrite": "小明把书给了小红，因为小红需要书。"
}
```

## 7. 实验评估方案

### 7.1 评价指标

可使用以下指标：

- Accuracy：代词指代预测正确率。
- Precision：系统预测的指代关系中有多少是正确的。
- Recall：真实指代关系中有多少被系统找出。
- F1：Precision 和 Recall 的综合指标。

### 7.2 对比实验

建议设计两组实验：

1. 纯规则方法。
2. 规则 + 语义匹配方法。

对比目标：

- 观察加入语义模型后是否提升准确率。
- 分析不同类型样本上的错误原因。

### 7.3 错误分析

报告中可以分析以下错误类型：

- 多个候选人物导致判断错误。
- 代词与先行词距离较远。
- 隐含语义需要常识判断。
- HanLP 实体识别错误。
- 改写后句子略显重复或不自然。

## 11. 风险与解决方案

### 风险一：中文指代消解难度较高

解决方案：

- 限定任务范围，优先处理人称代词和简单指示代词。
- 使用规则方法保证基础效果。
- 在报告中说明复杂语义样本属于未来改进方向。

### 风险二：HanLP 或模型安装较重

解决方案：

- 先实现不依赖深度模型的规则版本。
- 保留 HanLP 和 BERT 作为增强模块。
- 如果环境配置困难，可以用 jieba 和自定义词典替代。

### 风险三：改写文本不够自然

解决方案：

- 先采用简单替换策略。
- 对重复表达进行少量规则优化。
- 把改写模块定位为“可解释改写”，不追求生成式润色。
