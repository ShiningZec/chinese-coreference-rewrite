# 中文指代消解与句子改写数据标注规范

## 1. 标注目标

本数据集用于训练、测试和展示中文指代消解与句子改写系统。每条样本需要标注文本中的代词或指代表达，并给出它们对应的先行词。

示例：

```json
{
  "text": "小明把书给了小红，因为她需要它。",
  "coreference": [
    {"pronoun": "她", "antecedent": "小红"},
    {"pronoun": "它", "antecedent": "书"}
  ],
  "rewrite": "小明把书给了小红，因为小红需要书。"
}
```

## 2. 样本字段

### text

原始中文文本。建议每条样本包含 1-2 个句子，长度控制在 10-60 个汉字。

### coreference

指代关系列表。每个对象包含：

- `pronoun`：文本中的代词或指代表达。
- `antecedent`：该代词对应的先行词。

### rewrite

根据指代关系改写后的文本。改写原则是“明确指代对象”，不追求文学润色。

## 3. 标注对象

优先标注以下表达：

1. 人称代词：他、她、它、他们、她们、它们。
2. 组织指代表达：该公司、这家公司、该校、该团队。
3. 物体指代表达：这个产品、该产品、这本书、这个项目。
4. 地点指代表达：这里、该地、这个城市。

## 4. 先行词选择原则

1. 先行词应尽量选择文本中已经出现的最短明确实体。
2. 如果“华为公司”出现，不标成“华为”，优先保留原文完整实体。
3. 如果“新产品”出现，而规则系统只抽取“产品”，可以标注为“产品”，便于当前系统评估。
4. 如果存在多个候选实体，选择语义上最合理的一个。
5. 如果句子本身无法判断，不纳入正式测试集，可放入错误分析样本。

## 5. 改写原则

1. 直接将代词替换为先行词。
2. 尽量保持原句结构不变。
3. 不额外添加原文不存在的信息。
4. 允许改写结果略显重复，因为本项目强调可解释性。

示例：

```text
原文：小红买了一部手机，她很喜欢它。
改写：小红买了一部手机，小红很喜欢手机。
```

## 6. 推荐样本类型

数据集中建议覆盖以下类型：

1. 简单人物指代。
2. 简单物体指代。
3. 人物与物体同时出现。
4. 组织指代。
5. 学校、团队等机构指代。
6. 多候选人物。
7. 多候选物体。
8. 语义歧义样本。
9. 距离较远的指代。
10. 指示短语内部指代，例如“这个项目”。

## 7. 数据划分建议

期末作业阶段建议：

- `demo`：10 条，用于页面展示。
- `dev`：40 条，用于调规则。
- `test`：50 条，用于最终评估。

当前项目可以先将所有样本保存在 `data/samples.json`，后续再拆分为：

```text
data/demo.json
data/dev.json
data/test.json
```
